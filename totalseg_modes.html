TotalSegmentator — mode pipelines vs. total

Every mode runs through the same backbone (nnUNet_predict_image in totalsegmentator/nnunet.py). Modes differ only in configuration set by the dispatcher in totalsegmentator/python_api.py: which nnU‑Net task(s), what target spacing, which trainer, whether to crop with a rough pre‑segmentation, and a handful of mode‑specific post‑processing branches. This page ignores the --fast / --fastest / --higher_order_resampling / --roi_subset toggles and lists only the mode itself.

0. Shared backbone (what every mode does)

  1. Optional rough-seg pre-pass (only if the mode declares a crop ROI list or a crop_model). Runs total/total_mr/body at coarse spacing, then builds a binary mask from the listed ROI labels.
  2. Crop the input to that binary mask with a per-mode voxel margin (crop_addon).
  3. Reorient to closest canonical (LAS).
  4. Forward resample to resample mm, cubic spline on GPU (cucim, order=3).
  5. Optional Z-split into 3 sub-volumes (multi-model modes only, when the volume exceeds 512·512·900 voxels and Z > 200).
  6. nnU-Net predict with sliding window; tile step 0.8 only for plain CT total, otherwise 0.5.
  7. Merge multi-model outputs into one label volume (only multi-model modes: total, total_mr, headneck_muscles).
  8. Per-mode postprocess (only fires for body, appendicular_bones, kidney_cysts, face_mr, heartchambers_highres).
  9. Inverse resample with nearest-neighbour, undo canonical, undo crop.
  10. Save multilabel NIfTI (and for body: derived body.nii.gz + skin.nii.gz).

3. MR modes — diff vs. total

4. Behaviours exclusive to total

These are gated on the literal string task_name == "total" or on the multi-model shape. Even total_mr does not inherit them.

  • Sliding-window tile step 0.8 — only total. total_mr and every specialty mode use 0.5.
  • 5-part class merging via class_map_5_parts + map_taskid_to_partname_ct. total_mr uses a 2-part variant; headneck_muscles uses its own 2-part variant; everything else is single-model.
  • Triple Z-split is only enabled in multi-model modes (total, total_mr, headneck_muscles).
  • v1-label reordering (v1_order=True) — only total.
  • ROI-subset filtering is rejected unless task.startswith("total").

5. Recursion / cascade specials

  • teeth is the only mode whose pre-pass is a recursive full TotalSegmentator run (with task="craniofacial_structures") rather than the cheap rough total model.
  • The cascade input channel (a second nnU-Net input volume carrying a rough segmentation) is wired through the backbone but no enabled mode sets cascade=True; the only branch that did is the commented-out earlier total_highres_test.
  • heartchambers_highres is the only mode that activates the remove_outside_of_mask postprocess (with a 10 mm dilation).
