Root causes & fixes — IMPLEMENTED

Every sub-0.98 mode is now resolved to ≥0.999 parity. The gaps had three independent root causes, each isolated by capturing official's per-function intermediates (cropped model input s01_0000 and prediction s01) and bisecting where the pipelines diverge. All fixes run on GPU.
① Input-data resample: order-1 trilinear → order-3 cubic B-spline (GPU). The fast path resampled input data with F.interpolate(trilinear) (order-1); official nnU-Net's resample_data_or_seg uses skimage.resize(order=3) — a cubic B-spline. Isolation: scipy order-1 (0.872) ≈ torch trilinear (0.871); only order-3 reached 0.999 — so it is the interpolation order, not coordinate convention. PyTorch has no 3-D cubic, so we implemented a separable cubic B-spline on GPU (Unser/ITK recursive prefilter, pole √3−2; 4-tap sampling; 12-voxel edge prepad + grid-mode coords + per-channel clip) that reproduces scipy.ndimage.zoom(order=3,mode='nearest',grid_mode=True)+clip = resample_data_or_seg to ~1e-13, 7–60× faster than scipy. Fixes pleural_pericard_effusion 0.871→0.999 and lung_nodules 0.943→1.0.
② cucim input resample missing dtype=np.int32. Official truncates the resampled image to int32 before the model (change_spacing(...,dtype=np.int32), nnunet.py:485); we kept float. The fractional part diverged on low-signal MR edges and shifted the rough-seg crop mask. Adding dtype=np.int32 fixes total_mr 0.927→0.9999 (scapula/edge bones), liver_segments_mr 0.79→1.0, and the liver crop bbox.
③ Convert: threshold → softmax-argmax (per low-confidence mode). Default convert thresholded raw logits at 0.5; official uses label_manager softmax→argmax. Fine for confident organs, but it dropped low-confidence lesion voxels. Enabling use_softmax per-mode for liver_lesions / liver_lesions_mr → 1.0 (kept off elsewhere, no regression: total stays 1.0).
Whole pipeline is now GPU. Crop (crop_to_mask/undo_crop → torch bbox+slice) and connected-component postprocess (keep_largest_blob/remove_small_blobs/remove_outside → cupyx label/binary_dilation) were ported to GPU, bit-identical to the scipy originals. Plus cuDNN deterministic flags for reproducibility — the fp16 forward otherwise shifts a rough-seg crop mask by ~1 voxel across processes (e.g. liver_lesions 1.0↔0.95).
liver_lesions_mr is intrinsically nondeterministic, on both sides. Its 86-voxel lesion sits exactly on the rough-seg crop boundary; official itself flips between fg 86 and 52 across runs (we observed 86/52/86). Our deterministic output (fg 52) matches official's same-draw at DSC 1.0 — the apparent "0.56" was comparing our stable output to a different official coin-flip, not a parity defect.
Lung-vessel positive control (VESSEL12, n=10): lung_vessels 0.99998 and lung_vessels_LEGACY 0.99989 on near-isotropic VESSEL12 thoracic CT. (The Luna25 anisotropic 0.9913/0.7871 seen earlier was the order-1-vs-order-3 resample issue now fixed in ①.)
Recursive-crop validation (teeth, ToothFairy3, n=10): teeth (113) 0.99992 parity on dental CBCT — the last unvalidated mode, now confirmed. The full cascade total → craniofacial_structures → teeth runs end-to-end and matches official to ≥0.9997 on every case at 3.94× speedup. (`stage2_teeth_toothfairy3.csv`)
